# Contributing

## Scope
This project is a hackathon prototype. Contributions should improve stability, clarity, reproducibility, and documentation while preserving intent.

## Before You Start
1. Read [README.md](./README.md), [ARCHITECTURE.md](./ARCHITECTURE.md), and [SETUP.md](./SETUP.md).
2. Open an issue describing the bug, improvement, or refactor plan.
3. Keep pull requests focused on one concern.

## Development Setup
Use the instructions in [SETUP.md](./SETUP.md).

## Contribution Guidelines
1. Prefer small, reviewable commits.
2. Keep behavior changes explicit in PR descriptions.
3. Update docs when setup, architecture, or data behavior changes.
4. Avoid committing large generated artifacts unless explicitly required.
5. Follow existing project structure unless a restructuring proposal is approved.

## Pull Request Checklist
- [ ] Code builds and runs locally.
- [ ] New/changed behavior is documented.
- [ ] No accidental local artifacts are included.
- [ ] Any dataset or large-file changes are justified.
- [ ] PR description includes rationale and test notes.

## Reporting Issues
Please include:
- Environment details (OS, Node, Python versions)
- Exact reproduction steps
- Expected result and actual result
- Relevant logs, stack traces, or screenshots

## Code Of Conduct
By contributing, you agree to follow [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md).
